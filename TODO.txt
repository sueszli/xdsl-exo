# Tensor simplification ideas for bench_exojit.py

## Current usage analysis
- Tensor is only used for: allocation, view(), and .ptr access
- Indexing (__getitem__/__setitem__) is only used in init_normal_() and tokenize()
- The actual kernels receive raw pointers, not Tensor objects

## Ideas for simplification

### Option 1: Replace Tensor with a simple Buffer namedtuple
```python
Buffer = namedtuple("Buffer", ["ptr", "shape", "numel"])
```
- Just track ptr, shape, numel
- Use raw ctypes allocation functions
- Eliminates ~50 lines of Tensor class

### Option 2: Use numpy arrays directly
- Already imported, no new dependency
- np.ndarray has .ctypes.data for pointer access
- Built-in views and slicing
- Would need to handle dtype conversion (float64/int64)

### Option 3: Flat buffer + layout offsets only
- Keep a single flat ctypes array
- Precompute byte offsets for each "field"
- Pass (base_ptr + offset) directly to kernels
- Eliminates view() abstraction entirely

### Option 4: Minimal 1D ctypes array wrapper
```python
class Buf:
    __slots__ = ("ptr", "n")
    def __init__(self, n, dtype=float):
        ct = c_double if dtype is float else c_int64
        self._arr = (ct * n)()
        self.ptr = addressof(self._arr)
        self.n = n
```
- Shape tracking can be implicit (hardcoded constants)
- Views become just offset arithmetic on ptr

## Recommended approach
**Option 4** seems tightest - the shapes are all compile-time constants (BLOCK_SIZE, N_EMBED, etc).
We don't need dynamic shapes. Just:
- Raw ctypes arrays
- Precomputed offsets
- Pass (ptr + offset) to kernels

Could reduce from ~50 lines (Tensor class) to ~10 lines of helper functions.
