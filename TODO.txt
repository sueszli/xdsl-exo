TUTORIAL: implementing xdsl_exo/llvmlite_backend.py
=====================================================

Goal: a single self-contained file that takes the ModuleOp already produced by
compile_procs() and converts it to an llvmlite ir.Module, completing the pipeline:

    Exo -> xDSL (existing) -> llvmlite (this file)

The file has one public symbol:

    from xdsl_exo.llvmlite_backend import to_llvmlite

    llvm_mod = to_llvmlite(compile_procs(my_procs))
    print(llvm_mod)                   # LLVM IR text
    # or feed into llvmlite.binding for JIT

Nothing in the existing project needs to change. Add llvmlite to pyproject.toml:

    [project]
    dependencies = [
        ...
        "llvmlite>=0.43",
    ]

llvmlite bundles its own LLVM. No system dependency.


----------------------------------------------------------------------
WHAT THE INPUT LOOKS LIKE
----------------------------------------------------------------------

Run `uv run xdsl-exo examples/demo.py` to see what compile_procs() returns.
The module contains a mix of dialects that are NOT fully in LLVM yet:

    builtin.module {
      func.func @add(%0 : i64, %p0 : !llvm.ptr, %p1 : !llvm.ptr, %p2 : !llvm.ptr) {
        %c0 = arith.constant 0 : i64
        %c1 = arith.constant 1 : i64
        cf.br ^bb0(%c0 : i64)
      ^bb0(%i : i64):
        %cmp = arith.cmpi slt, %i, %0 : i64
        cf.cond_br %cmp, ^bb1, ^bb2
      ^bb1:
        %idx = arith.index_cast %i : i64 to index
        %bpe = arith.constant 4 : index
        %off = arith.muli %idx, %bpe : index
        %off64 = arith.index_cast %off : index to i64
        %p0i = "llvm.ptrtoint"(%p0) : (!llvm.ptr) -> i64
        %p0o = arith.addi %p0i, %off64 : i64
        %p0p = "llvm.inttoptr"(%p0o) : (i64) -> !llvm.ptr
        %v0  = "llvm.load"(%p0p) : (!llvm.ptr) -> f32
        ... (similar for p1)
        %sum = arith.addf %v0, %v1 : f32
        ... (store sum, increment i, loop back)
        cf.br ^bb0(%i_next : i64)
      ^bb2:
        func.return
      }
      func.func @noop() { func.return }
      llvm.func @malloc(i64) -> !llvm.ptr
      llvm.func @free(!llvm.ptr)
    }

The ops that xDSL's own llvm backend (xdsl.backend.llvm.convert) does NOT handle:

    func.func      func.return
    arith.constant arith.addi  arith.addf  arith.muli
    arith.cmpi     arith.index_cast
    cf.br          cf.cond_br
    multi-block functions (xDSL backend hard-raises NotImplementedError)

Everything that IS already llvm.* is handled correctly by xDSL's convert_op.
We reuse it as a fallback rather than reimplementing it.

IMPORTS
----------------------------------------------------------------------

    import llvmlite.ir as ir

    from xdsl.backend.llvm.convert_op import convert_op    # handles llvm.* ops
    from xdsl.backend.llvm.convert_type import convert_type # handles all types
    from xdsl.dialects import arith, cf, func, llvm
    from xdsl.dialects.builtin import FloatAttr, IntegerAttr, ModuleOp
    from xdsl.ir import Block, SSAValue

convert_type already handles: IntegerType, Float{16,32,64}Type, LLVMPointerType,
VectorType, LLVMArrayType, LLVMStructType, IndexType -> i64. Use it everywhere.


----------------------------------------------------------------------
FUNCTION 1: _emit_arith
----------------------------------------------------------------------

Handles arith.* ops that appear in the pipeline output.

Verified field access (from actual compiled output):

    arith.ConstantOp:
        op.value              -> IntegerAttr or FloatAttr
        op.value.value.data   -> Python int or float (the literal value)
        op.value.type         -> the xDSL type (e.g. IntegerType(64), IndexType())
        op.result             -> the SSAValue to register in val_map

    arith.AddiOp / MuliOp / AddfOp:
        op.lhs, op.rhs        -> SSAValue operands
        op.result             -> SSAValue result

    arith.CmpiOp:
        op.lhs, op.rhs        -> SSAValue operands
        op.predicate          -> IntegerAttr, .value.data is an int (see table below)
        op.result             -> SSAValue result

    arith.IndexCastOp:
        op.operands[0]        -> SSAValue operand
        op.results[0]         -> SSAValue result
        (index == i64 on 64-bit; this is a no-op cast)

Predicate table for arith.cmpi (.predicate.value.data -> (llvmlite_op, signed)):

    0  eq   ("==",  True )    builder.icmp_signed
    1  ne   ("!=",  True )
    2  slt  ("<",   True )    <- this is what the demo uses
    3  sle  ("<=",  True )
    4  sgt  (">",   True )
    5  sge  (">=",  True )
    6  ult  ("<",   False)    builder.icmp_unsigned
    7  ule  ("<=",  False)
    8  ugt  (">",   False)
    9  uge  (">=",  False)

Sketch:

    _CMPI_PRED = {
        0: ("==", True), 1: ("!=", True),
        2: ("<",  True), 3: ("<=", True), 4: (">", True), 5: (">=", True),
        6: ("<",  False), 7: ("<=", False), 8: (">", False), 9: (">=", False),
    }

    def _emit_arith(op, builder, val_map):
        match op:
            case arith.ConstantOp():
                ty  = convert_type(op.value.type)
                val = op.value.value.data
                val_map[op.result] = ir.Constant(ty, val)

            case arith.AddiOp() | arith.MuliOp():
                # index-typed operands need the same treatment as i64;
                # convert_type(IndexType()) returns ir.IntType(64)
                a, b = val_map[op.lhs], val_map[op.rhs]
                fn = builder.add if isinstance(op, arith.AddiOp) else builder.mul
                val_map[op.result] = fn(a, b)

            case arith.AddfOp():
                val_map[op.result] = builder.fadd(val_map[op.lhs], val_map[op.rhs])

            case arith.CmpiOp():
                pred_int = op.predicate.value.data
                llvm_pred, signed = _CMPI_PRED[pred_int]
                fn = builder.icmp_signed if signed else builder.icmp_unsigned
                val_map[op.result] = fn(llvm_pred, val_map[op.lhs], val_map[op.rhs])

            case arith.IndexCastOp():
                # index == i64 on 64-bit targets; treat as identity
                # if source and target types differ (index <-> i64), emit a bitcast;
                # in practice convert_type gives the same ir.IntType(64) for both
                src = val_map[op.operands[0]]
                dst_ty = convert_type(op.results[0].type)
                if src.type == dst_ty:
                    val_map[op.results[0]] = src          # true no-op
                else:
                    val_map[op.results[0]] = builder.bitcast(src, dst_ty)

            case _:
                raise NotImplementedError(f"unhandled arith op: {op.name}")

NOTE on IndexType constants: arith.constant 4 : index produces
IntegerAttr(value=IntAttr(data=4), type=IndexType()). The value.data is still
a plain Python int. convert_type(IndexType()) gives ir.IntType(64). This works
correctly with the Constant constructor.


----------------------------------------------------------------------
FUNCTION 2: _emit_cf
----------------------------------------------------------------------

Handles cf.br and cf.cond_br. This is the trickiest part because MLIR uses
block arguments instead of phi nodes, and llvmlite uses the traditional phi form.

Phi nodes were created in pass 1 of _convert_func (see below) and are stored in:

    phi_map: dict[BlockArgument, ir.PhiInstr]

At branch time, we know the source block and the values being passed, so we can
call phi.add_incoming() immediately. No deferred fixup list needed.

Verified field access:

    cf.BranchOp:
        op.successor          -> target Block
        op.successor.args     -> list of BlockArgument (the phi owners)
        op.operands           -> list of SSAValue (values to pass; same length as args)

    cf.ConditionalBranchOp:
        op.cond               -> SSAValue (the i1 condition)
        op.successors[0]      -> true Block
        op.successors[1]      -> false Block
        op.then_arguments     -> tuple of SSAValue (args for true branch)
        op.else_arguments     -> tuple of SSAValue (args for false branch)
        (in this project's output both are empty, but handle generally)

Sketch:

    def _emit_cf(op, builder, block_map, phi_map, val_map):
        current_llvm_block = builder.block

        match op:
            case cf.BranchOp():
                target = block_map[op.successor]
                # add_incoming for each block argument of the target
                for block_arg, operand in zip(op.successor.args, op.operands):
                    if block_arg in phi_map:
                        phi_map[block_arg].add_incoming(val_map[operand],
                                                        current_llvm_block)
                builder.branch(target)

            case cf.ConditionalBranchOp():
                true_bb  = block_map[op.successors[0]]
                false_bb = block_map[op.successors[1]]
                # then_arguments / else_arguments are the args passed to each branch
                for block_arg, operand in zip(op.successors[0].args, op.then_arguments):
                    if block_arg in phi_map:
                        phi_map[block_arg].add_incoming(val_map[operand],
                                                        current_llvm_block)
                for block_arg, operand in zip(op.successors[1].args, op.else_arguments):
                    if block_arg in phi_map:
                        phi_map[block_arg].add_incoming(val_map[operand],
                                                        current_llvm_block)
                builder.cbranch(val_map[op.cond], true_bb, false_bb)

            case _:
                raise NotImplementedError(f"unhandled cf op: {op.name}")


----------------------------------------------------------------------
FUNCTION 3: _emit_op (dispatch)
----------------------------------------------------------------------

Central dispatch. Falls back to xDSL's convert_op for all llvm.* ops.
func.return and llvm.ReturnOp are structurally identical so handle both here.

    def _emit_op(op, builder, block_map, phi_map, val_map):
        match op:
            case arith.ConstantOp() | arith.AddiOp() | arith.MuliOp() \
               | arith.AddfOp() | arith.CmpiOp() | arith.IndexCastOp():
                _emit_arith(op, builder, val_map)

            case cf.BranchOp() | cf.ConditionalBranchOp():
                _emit_cf(op, builder, block_map, phi_map, val_map)

            case func.ReturnOp() | llvm.ReturnOp():
                if op.operands:
                    builder.ret(val_map[op.operands[0]])
                else:
                    builder.ret_void()

            case _:
                # delegate everything else (all llvm.*) to xDSL's backend
                convert_op(op, builder, val_map)

If you add more intrinsics later, add cases before the fallthrough.
The fallthrough will raise NotImplementedError for any unrecognised op,
which is the right failure mode.


----------------------------------------------------------------------
FUNCTION 4: _convert_func  (the core)
----------------------------------------------------------------------

Works for both func.FuncOp and llvm.FuncOp — they share the same structure:
    .sym_name.data          -> str function name
    .function_type.inputs   -> list of xDSL types (arg types)
    .function_type.output   -> xDSL type (return type; use .outputs[0] or VoidType)
    .body.blocks            -> iterable of Block

Body-less ops (extern declarations like malloc/free) are llvm.FuncOp with no body.
Guard on that first; func.FuncOp always has a body in this project.

The two-pass approach:

    PASS 1 — allocate llvmlite basic blocks and phi nodes
    -------------------------------------------------------
    For each MLIR block (in order):
      - append a new llvmlite basic block
      - store in block_map: Block -> ir.Block

    For each MLIR block that is NOT the entry block:
      - for each block argument:
          position a temporary builder at the start of the llvmlite block
          create a phi node with the correct type
          store in phi_map: BlockArgument -> ir.PhiInstr
          store in val_map: BlockArgument -> ir.PhiInstr   (same object)

    For the entry block:
      - block args are function parameters; map func.args -> val_map directly
        (llvmlite gives these via func.args, same order as function_type.inputs)

    PASS 2 — emit instructions
    --------------------------
    For each MLIR block (in order):
      - position builder at that llvmlite block
      - for each op in block:
          _emit_op(op, builder, block_map, phi_map, val_map)

    The phi.add_incoming() calls happen inside _emit_cf during pass 2,
    at the point where branch ops are encountered.

Sketch:

    def _convert_func(func_op, llvm_module):
        # skip extern declarations
        if not func_op.body.blocks:
            return

        # build llvmlite function type
        output_types = list(func_op.function_type.outputs)
        ret_ty  = convert_type(output_types[0]) if output_types else ir.VoidType()
        arg_tys = [convert_type(t) for t in func_op.function_type.inputs]
        ftype   = ir.FunctionType(ret_ty, arg_tys)
        func    = ir.Function(llvm_module, ftype, name=func_op.sym_name.data)

        mlir_blocks = list(func_op.body.blocks)
        block_map: dict[Block, ir.Block]         = {}
        phi_map:   dict[SSAValue, ir.PhiInstr]   = {}
        val_map:   dict[SSAValue, ir.Value]       = {}

        # pass 1a: create llvmlite blocks
        for mlir_block in mlir_blocks:
            llvm_block = func.append_basic_block()
            block_map[mlir_block] = llvm_block

        # pass 1b: map entry block args -> function params
        entry = mlir_blocks[0]
        for mlir_arg, llvm_arg in zip(entry.args, func.args):
            val_map[mlir_arg] = llvm_arg

        # pass 1c: create phi nodes for non-entry block args
        for mlir_block in mlir_blocks[1:]:
            if not mlir_block.args:
                continue
            llvm_block = block_map[mlir_block]
            tmp_builder = ir.IRBuilder(llvm_block)
            for block_arg in mlir_block.args:
                phi = tmp_builder.phi(convert_type(block_arg.type))
                phi_map[block_arg] = phi
                val_map[block_arg] = phi

        # pass 2: emit ops
        for mlir_block in mlir_blocks:
            llvm_block = block_map[mlir_block]
            builder = ir.IRBuilder(llvm_block)
            for op in mlir_block.ops:
                _emit_op(op, builder, block_map, phi_map, val_map)


----------------------------------------------------------------------
FUNCTION 5: to_llvmlite  (public entry point)
----------------------------------------------------------------------

Iterates top-level ops in the module. The module contains:
    func.FuncOp  (user procedures — have bodies)
    llvm.FuncOp  (extern declarations — no bodies: malloc, free)

Both are handled by _convert_func via the body-check guard.

    def to_llvmlite(module: ModuleOp) -> ir.Module:
        llvm_module = ir.Module()
        for op in module.ops:
            match op:
                case func.FuncOp() | llvm.FuncOp():
                    _convert_func(op, llvm_module)
                case _:
                    raise NotImplementedError(
                        f"to_llvmlite: unhandled top-level op: {op.name}"
                    )
        return llvm_module

The return value is an ir.Module. str(llvm_module) gives LLVM IR text.
To JIT it, use llvmlite.binding (llvm.create_mcjit_execution_engine etc.).

----------------------------------------------------------------------
KNOWN EDGE CASES TO WATCH
----------------------------------------------------------------------

1. IndexType constants: arith.constant 4 : index
   -> op.value.type is IndexType(), not IntegerType(64)
   -> convert_type(IndexType()) should return ir.IntType(64)
   -> verify this holds; if not, special-case it in _emit_arith

2. index-typed operands in arith.addi / arith.muli
   -> both operands of arith.muli are IndexType in the pointer arithmetic pattern
   -> convert_type must agree on the same ir.IntType(64) for both lhs and rhs
   -> llvmlite will reject a binary op if lhs.type != rhs.type

3. Entry block args are function params, NOT phi nodes
   -> Do NOT create phi nodes for mlir_blocks[0].args
   -> Map them from func.args directly in pass 1b

4. Body-less llvm.FuncOp (extern declarations)
   -> func_op.body.blocks is empty -> early return from _convert_func
   -> llvmlite does NOT need explicit extern declarations for malloc/free;
      llvm.call will reference them by name if they were declared elsewhere,
      but in llvmlite's ir.Module they need to exist as ir.Function objects
      with no body. Check whether xDSL's llvm.CallOp handler calls
      builder.module.get_global(name) — if the global doesn't exist it raises.
      Solution: declare extern functions as ir.Function with no blocks appended.

5. phi.add_incoming timing
   -> phi nodes must have at least one incoming edge before the IR is printed
   -> if a block is only reachable via one branch, it gets exactly one incoming
   -> the loop-back block (^bb0 in the demo) gets two: the initial branch and
      the back-edge. Both are added during pass 2 as each cf.br is processed.

6. Builder positioning after a terminator
   -> ir.IRBuilder raises if you try to emit after a branch/ret
   -> In MLIR, each block ends with exactly one terminator (cf.br, cf.cond_br,
      func.return); so this should not be an issue if the input is well-formed
   -> module.verify() before calling to_llvmlite guarantees well-formedness
