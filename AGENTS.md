# Code Style Guidelines

## Formatting
- DO NOT reformat code when making edits
- Make minimal changes - only modify what's necessary
- Match the existing code style of surrounding code
- Run `make fmt` after completing edits if formatting is needed

## Line Length
- This project uses very long lines (5000 char limit)
- Do not wrap or break lines

## Imports
- Do not reorganize imports
- Run `make fmt` to handle import sorting automatically
